## 2. Alpha-derived prop footprints (THE failure mode — fixed)

Footprints were derived from each PNG's **alpha channel** (8-connected components,
bbox snapped out to whole 16px tiles), NOT fixed GID blocks. Verified by rendering
the actual tileset art over a forest crop: every tree is a complete unit, no
half-trees, no fruit-fragment-in-canopy artifacts. The single-tile fruit **icons**
(item drops) were detected as their own 1×1 stamps and **excluded** from world placement.

Local-tile footprints used (local id = row×cols + col):

**`trees_stumps_bushes.png` (12×7)** — colliding trees & understory decor:
- plain tree **2×2** = locals 1,2,13,14 (also the `baseTree` for fruit trees)
- small tree **1×2** = locals 0,12
- hero tree **3×3** = locals 45,46,47 / 57,58,59 / 69,70,71 (deep-forest anchor, sparse)
- fruit overlays **2×2** (same footprint as plain): apple 3,4,15,16 · orange 5,6,17,18 · yellow 7,8,19,20 · pink 9,10,21,22
- bush **1×1** = local 36 · stump **1×1** = local 72 · logs **2×1** = 74-75 / 79-80 / 81-82
- EXCLUDED: fruit icons (locals 24–31, the 1×1 item-drop row)

**`mushrooms_flowers_stones.png` (12×5)** — understory + dynamic flora:
- mushroom_red frames (1×1) = locals 0,1,2 · mushroom_purple = 3,4,5
- pebbles (1×1) = 12,13,14,15 · boulder **2×2** = 18,19,30,31 (sparse) · grass tufts (1×1) = 24,25,26,27
- flower_yellow frames = 36,37,38 · flower_pink = 40,41,42 · flower_blue = 52,53,54 · sunflower **1×2** = 39,51 (landmark)

**`Water Objects.png` (12×2)** — `props_water`, water tiles only:
- water rocks (1×1) = 0,1,2,3 · big rock **2×1** = 4,5 · reeds (1×1) = 6,7 · lily pads (1×1) = 8,9(flowering),10
- bottom-row pond-fill blobs intentionally unused (real water tiles preferred, per spec)

Placement safety: every stamp is checked so its full footprint lands on valid
ground (not water/path/homestead/another prop) — whole, or skipped. An `occupied`
set guarantees zero prop overlaps.

## 3. Two-tier density

- **Colliding tier (sparse):** tree trunks + hero trees, placed on `props_trees`.
  Minimum trunk-to-trunk gap enforced: meadow 5, light 4, mid 3, deep 3 tiles —
  the player can always thread through. Trunk placement probabilities: meadow 0.012,
  light 0.05, mid 0.09, deep 0.13.
- **Non-colliding tier (dense):** flowers, mushrooms, grass tufts, pebbles, bushes,
  logs, boulders on `props_ground` (walk-through). Densities: meadow 0.10, light 0.16,
  mid 0.22, deep 0.30 — deep forest reads lush without blocking movement.

Deep forest is concentrated toward the map perimeter via a radial biome field
(meadow core → light → mid → deep) plus scattered meadow pockets.

## 4. Path grammar (Stone_Path vs Paths) — confirmed

- **`Stone_Path.png` → `paths_main` (arterials):** width-3 routes leaving the
  homestead's N / E / W gates straight to the map edges — the backbone network.
- **`Paths.png` (wooden) → `paths_spur` (capillaries):** width-2 spurs that branch
  off arterials and **terminate at a payload** — a lake/seed spot, an encounter
  clearing, or a deliberate deep-forest dead-end (see the `markers` layer `payload`
  objects). Never aimless.
- **`Wooden_Bridge_v2.png` → `bridges`:** auto-placed at every path×water crossing,
  oriented horizontal vs vertical from the local path direction. Crossings are kept
  plentiful for traversability (5 on the 150 maps, 10 on the 400) — remove extras in Tiled if desired.

## 5. Homestead — extraction method

The live homestead is **procedural** (`GameScene.js`: `createGardenFence`,
`spawnGardenBeds`, `spawnGardenStructures`), not tilemap data, so it cannot be
copied as tiles directly. Per the spec's fallback clause, its **layout footprint
was reproduced** from the procedural constants:

- 50×50-tile fenced square (matches the live `GARDEN_WIDTH/HEIGHT = 800px`), centered
  per map (150 → 75,75; 400 → 200,200) — same absolute size on every map.
- 4-sided fence (`Fences.png`) with **N/E/W gate openings** (`fence_gate.png`); the
  **south side is solid** (kept consistent with the cliff-backed homestead design).
- 8 garden beds (2 rows × 4) as 2×2 tilled-soil patches (`Soil_Ground_Tiles.png`).
- Well (`water_well.png`, 2×2) left-interior; work station (`work_station.png`, 2×2)
  right-interior.
- The `markers` object layer carries `player_start`, `gate_north/east/west`, `well`,
  `work_station`, and the 8 `garden_bed` positions for the engine to read.

Substitution flag: bed/well/work-station **positions** are a faithful reproduction
of the dev layout's relative geometry, not a byte-for-byte extraction. Redesign
freely in Tiled — the engine should read the markers, not hard-code positions.

## 6. Living-world data model + draw logic

`nature_dynamic` object-layer entries (each also has its frame-0 sprite painted into
`props_ground`/`props_trees` as the static fallback):
- **flower / mushroom:** `kind`, `species`, `frames` (CSV of growth-stage GIDs),
  `offset` (random 0..frames-1 per instance → no unison pulsing).
  Runtime frame = `frames[(dayCount + offset) % frames.length]`.
- **fruit_tree:** `kind`, `baseTree` (plain-tree GIDs), `fruitType`
  (apple|orange|yellow|pink), `fruitOverlay` (fruited GIDs). Global season decides
  fruiting: `season = ['apple','orange','yellow','pink'][floor(dayCount/SEASON_LENGTH)%4]`.

`src/world/DynamicNature.js` implements §6: reads `nature_dynamic`, computes
`currentSeason` and reads the existing day counter (marked `// SEASON/DAY CYCLING HOOK`),
draws the correct per-day frame, and is **static-safe** — if no Tiled map / object
layer / tileset textures are loaded (the live build is still procedural), `init()`
no-ops with zero errors and the painted frame-0 sprites remain. It refreshes on the
project's `day:advanced` event when available. Nothing in combat/economy/save/homestead
is touched.

**Wiring (additive, 2 lines in `GameScene.js`):**
```js
import DynamicNature from '../world/DynamicNature.js';   // after the economyData import
// ...near the end of create(), after maybeSpawnDailySeed():
this.dynamicNature = new DynamicNature(this).init();
```
The likely future integration point for actually loading these maps is the existing
`feature/load-tiled-world` branch.
